Статистика

Верификации, прошедшие без показа рекламы — органическая активность серверов. Помогает оценить, сколько заходов (stays) можно продать следующим заказом.

Активность верификаций

Все серверы с активной карточкой, в реальном времени. Три стадии воронки идут отдельными линиями — любую можно скрыть. Серверы ниже фильтруют график; если оставить включённой одну стадию, выбранные серверы разложатся отдельными линиями для сравнения.

Заглушка

По серверам

Стата рекламы

Каждая карточка — уникальный креатив (готовый рендер текста с подставленной ссылкой). Учитываются только верификации, где реклама реально была показана; закрытый кран или клики без рекламы в счёт не идут.

Активность

Все события верификаций по всем партнёрам: начисления, списания, выдача и снятие верификации — включая те, за которые не было оплаты, сразу с причиной.

Доли

1% доли = 1% чистого дохода сервиса. Доход = продажа заходов ($10 / 100 с проверкой на заход) минус выплата партнёрам и эквайринг (3% от выплат). Чистая прибыль каждого захода делится по долям и начисляется на баланс — дальше обычная выплата (запрос на вывод от $10 или автовывод).

Карточки верификации

Список всех карточек. Если карточка сломалась или работает некорректно — «Встряхнуть» пересоберёт её на месте (владелец и роль сохранятся). Также можно сменить владельца/роль, перепубликовать или удалить. Чтобы добавить карточку, созданную до отслеживания — вставьте ссылку на сообщение (ПКМ по карточке → «Копировать ссылку на сообщение»).

Удалённые карточки

Карточки, удалённые с канала (вручную, ботом или через панель). Статистика сохраняется; видно, кто и когда убрал верификацию (кто — если бот успел заметить удаление через аудит-лог).

Аудит-лог действий

Лента серверов

Серверы в ленте «Сообщества, которые уже зарабатывают с нами» на главной. Добавьте по ссылке-приглашению — имя, иконка и число участников подтянутся автоматически.

Настройки

Переменные окружения, которые можно менять без Railway. Все они применяются сразу по «Сохранить» — перезапуск не нужен (кнопка ниже осталась на всякий случай). Корневые секреты (токены ботов, владелец, OAuth) недоступны намеренно.

Лоты

Аукционы на количество stays. При запуске бот создаёт канал на аукционном сервере, где участники ставят цену числом в чат. Если ставку никто не перебьёт 15 минут — она побеждает и лот закрывается. История и результаты — ниже.

Название канала

Имя канала, который бот создаёт при запуске лота. Плейсхолдер: {stays} — кол-во заходов в лоте (как и в сообщении). Discord сам приводит к нижнему регистру и заменяет пробелы дефисами.

Сообщение при запуске

Текст, который бот отправляет в канал при старте лота. Плейсхолдеры: {stays} — кол-во stays, {sb} — начальная цена, {ob} — мин. надбавка к ставке. Работает Markdown Discord.

Исключение участников сервера

Необязательно: если указать сервер, его участники из guild_members не попадут в очередь рассылки. Без ID исключения не применяется.

Кому отправлять

Все участники, только роли или конкретные ID. Исключение по ролям и участникам применяется после фильтра аудитории. Дедуп по ссылке — отдельно, в блоке выше.
