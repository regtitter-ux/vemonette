Monetize your Discord bot with Vemoni

Vemoni is a Discord ad network. Add a few calls to your bot, offer your users a reward for joining a sponsor server, and earn $5 for every 100 joins.

After they join, call POST /api/join-check. We confirm it with Discord and credit you — then you hand over the reward.

The base URL and your key

Every endpoint sits under one base URL. Send your key with each request — either as a bearer token or in the X-API-Key header. Because calls come from your server, the key never touches a browser.

# Base URL
https://secure-gratitude-production.up.railway.app

# Send your key one of these two ways
Authorization: Bearer <your-key>
X-API-Key: <your-key>

A working example in one screen

Here is the whole reward loop (Node.js). userId is your user's Discord ID; serverId is a guild your bot is in — it decides which sponsor to show.

const BASE = 'https://secure-gratitude-production.up.railway.app';
const h = { 'Authorization': 'Bearer ' + process.env.VEMONI_KEY, 'Content-Type': 'application/json' };

// 1) Ask which sponsor to show, then show it to your user
const ad = await fetch(`${BASE}/api/ad?serverId=${serverId}`, { headers: h }).then(r => r.json());
if (ad.sponsor) showToUser(ad.sponsor.name, ad.sponsor.invite, 'Join for +100 coins!');

// 2) When they say they joined, confirm it — we check Discord and pay you
const out = await fetch(`${BASE}/api/join-check`, {
  method: 'POST', headers: h,
  body: JSON.stringify({ userId, serverId })
}).then(r => r.json());

if (out.joined) grantBonus(userId, 100); // confirmed → give your reward
else tellUser('Join the server first, then press Check.');

The endpoints, one by one

GET/api/ad

Add ?serverId=<guild>. Returns the sponsor to show: { sponsor: { guildId, name, invite } | null, adText, fallbackText }. If sponsor is null, there is nothing to promote right now — skip the reward and let the user through.

POST/api/join-check

Send { userId, serverId }. We ask Discord whether the user is in the current sponsor and answer: 200 { joined:true, credited:true } — confirmed and you have been paid, so give the reward; 403 { joined:false } — not in yet, ask them to join; 200 { joined:true, ad:false } — no sponsor was running, so no payout. Each member counts once per sponsor, and if they leave, the credit is reversed.

GET/api/balance

Your current balance and the payout details on file.

GET/api/stats

Your confirmed joins, broken down by server and by time window.

GET/api/requisites PUT

See or change where the money goes. PUT { requisites } — a crypto wallet, a Telegram ID for automatic USDT, and so on.

GET/api/withdrawals POST /api/withdraw

Your withdrawal history, plus a way to nudge the payout check. Once your balance reaches $10, a request is filed for you automatically.

Safe for you, safe for us

Only real joins pay

A join earns only after we confirm the membership with Discord itself. Each member counts once per sponsor, and leaving reverses the credit — so there is nothing worth faking.

You decide the reward

The network picks the sponsor and the payout rate; the reward your users get is entirely yours to choose — coins, roles, XP, anything that fits your bot.

Your key, your money

Keys are secret and can be regenerated any time. Earnings sit on your Discord account, and you set the payout details and withdraw from the cabinet.

No private data changes hands

You only ever send a Discord user ID and a guild ID. We never ask for message content, tokens, or anything private about your users.
